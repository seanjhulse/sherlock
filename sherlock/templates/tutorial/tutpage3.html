SherLOCK - Brief Introduction to Network Protocols


To begin deciphering the meaning behind the Nodemap, we must first begin by understanding what the individual components mean.

First, let's address the numbers that appear on the edges of the Nodemap.
These are port numbers. They describe what ports on a server are active entrances and exits for incoming/exiting data. Certain ports are assigned to perform different functions.
The most common port numbers you can expect to encounter while using SherLOCK are:

 - 80
 - 443

Port 80 is by default used for HTTP (Hyper Text Transfer Protocol) connections. Port 443 is used for HTTPS (Hyper Text Transfer Protocol Secure) connections.

Where you'll see a connection between the host node "This Computer" and some other node "server-13-226-234-70.lax50.r.cloudfront.net" with two curves connecting the two nodes. Port 443 represents the transfer of data from the server to the local host (you) and the randomly assigned port number represents the return transfer of data from you to the server.

If you look towards the upper left-hand corner of the Nodemap, you will notice a legend with some text in it.



This legend describes the different types of data packets we can expect to receive when using the Nodemap to observe network traffic.

The most common forms of Internet Protocols you will experience are TCP (Transmission Control Protocol) and UDP (User Datagram Protocol).
These two protocols are the most prevalent forms of data transfer. Both protocols are used to stream data, but the major difference between them is their execution.


TCP is connection-oriented. Meaning, that data does not begin to flow until a connection between streaming server and end-user (or client) is established. It is also checked for errors which aids reliability.



UDP, in contrast, operates by constantly streaming data before a connection between server and client is established and is not checked for correctness or errors. UDP is ideal for applications where correctness and error checks are not needed.

The vast majority of applications such as email, video-streaming, and online shopping sites operate off of TCP. As such, you will commonly see the Nodemap flash a green color as data packets are sent and received along TCP routes.
